如何使用官方插件

在RPG 制作大师 MZ中，新建的项目中已经包含了几个插件，可以通过简单的设定来使用。另外，某些官方插件单独包含在其它用途的DLC文件夹中。

AltMenuScreen

这是一个将菜单画面更改为其他布局的插件。
队伍成员将横向并排显示。

AltSaveScreen

这是一个将保存画面和加载屏幕更改为不同布局的插件。
可以显示队伍成员的面部图像等。

ButtonPicture

该插件在单击图片时会提供调用公共事件的指令。
请按以下顺序使用。

1.执行[显示图片]以显示按键图像。
2.调用插件指令[设置按键图片]。

EventCommandByCode

您可以通过直接指定代码和参数来执行事件指令。
变量（控制字符）可以用作各个指令的参数，并且可以强制指定正常事件指令无法指定的范围之外的值。
另外，如果调用的命令设置了意外的值，则不能保证实现操作。

控制字符

\v[n] : 变量值
\s[n] : 开关
\ss[n] : 独立开关
\js[n] : 脚本（仅限适用TextScriptBase.js时）

您无法在事件编辑器中指定复数行的指令或是带有深层嵌套的命令。

ExtraImage

您还能添加可以指定任意图像的场景。
您可以指定例如坐标、放大倍数、原点等基本信息。
如果为每个参数指定一个控制字符，则可以从指定的变量中获取值。

参数

图像列表 要添加到每个画面的图像列表。

ExtraWindow

您还能添加可以指定任意窗口的场景。
您可以设定例如坐标、字体大小、开闭动画等基本信息。
控制字符可以在窗口中显示的文本中使用，并且在变量值更改时会自动重绘。

参数

窗口列表 要添加到每个画面的窗口列表。

MaterialBase

这是一个可让您登录和管理主要由插件使用的图像和音频素材的数据库。
登录的素材会自动从删除未使用的素材的功能的目标中排除。
如果使用以下控制字符，则将替换已登录素材的文件名。

¥mi[aaa] // 用识别码[aaa]替换登录的图像素材的文件名。
¥mi[bbb] // 用识别码[bbb]替换登录的声音素材的文件名。

可以使用控制字符的情况如下。
・文字的显示
・事件指令或条件分支的脚本
・备注栏（※）
・插件指令（※）
・插件参数（※）
・技能等的说明栏。
・技能、物品的计算公式
※仅限使用PluginCommonBase.js为基础导入的插件

可以通过插件指令将登录的素材更改为其他素材。

设置的值可以通过插件指令作为[图片]或[BGM]显示、播放。
详细信息，请参见插件指令的说明。

另外，如果使用另一个插件[事件指令的代码执行插件]，则可以将其用于更广泛的用途。

参数

图像素材列表 登录的图像素材列表。
声音素材列表 登录的声音图像列表。

OverpassTile

可以在地图上显示例如桥梁等立体交叉结构。
这是制作大师 MV的官方插件“ OverpassTile.js”的面向MZ的功能增强版本。

不仅可以指定区域，还可以指定地形标签，并考虑了事件的启动、冲突判定等有关情况。
不会考虑载具的存在。

与允许通行有关的判定规范

・如果您在桥的入口处
→面对立体交叉必定可以移动。
・如果您在立体交叉结构时
→在桥上时，面对立体交叉和桥的入口时必定可以移动。
・如果您在桥下时，面对桥的入口必定无法进行移动。
・在上述条件下无法判断能否移动时
→遵循地图的原始通行设置。

您可以考虑事件的启动判定、冲突判定等因素。

・高度不同（桥的上方和下方）的事件不会启动或冲突。

参数

立体交叉区域 要在桥梁等立体交叉部分设置的区域ID。
立体交叉地形标签 要在桥梁等立体交叉部分设置的地形标签。
立体交叉入口区域 设置在例如桥梁的两端、立体交叉的入口部分的区域ID。
立体交叉入口地形标签 设置在例如桥梁的两端、立体交叉的入口部分的区域ID。

PluginCommonBase

本插件是以会被其他插件引用为前提设置的基本插件。
引入此插件后，可以在各种情况下使用以下控制字符。

可以使用的控制字符

\v[n] : 转换为第n个变量的值。
\s[n] : 转换为第n个开关的值(true, false)。
\ss[n] : 转换为第n个独立开关的值(true, false)。

可以使用控制字符的情况如下。

・文字的显示
・备注栏（※）
・插件指令（※）
・插件参数（※）
・技能等的说明栏。
※仅限使用PluginCommonBase.js为基础导入的插件

请参阅插件中的帮助以获取面向开发人员的说明。

参数

禁用遮罩处理 窗口重叠时禁用遮罩处理。

RegionBase

提供区域和地形标签等的数据库。
规格大致以RPG 制作大师 MV Trinity为基准。
通过将区域和地形标签用作触发器，可以提供以下功能。
・对事件和玩家进行通行判定（包括4个方向）
・梯子，灌木，柜台，损坏的地板
・调用公共事件（3种触发器）
・ 仅在入侵时才打开的开关
・入侵时才有效的特征
・备注栏

可使用以下脚本从脚本、外部插件等引用数据库。
请注意，如果未设置内容或后缀为[0]时，则内容将设为undefined。

$dataSystem.regions[ID];
$dataSystem.terrainTags[ID];

参数

区域列表 区域数据的列表。
地形标签列表 地形标签数据的列表。

TextPicture

该插件提供了将文本作为图片显示的命令。

请按以下顺序使用。
1.调用插件指令[文本图片的设置]。
2. 执行[图片的显示]而不指定图像。

TextScriptBase

这是一个可以登录和管理例如文本和脚本等多行字符串的数据库。
可以使用以下控制字符引用登录的文本。
¥tx[aaa] // 用识别码[aaa]替换已登录的文本。
¥js[bbb] // 用识别码[bbb]将已登录的文本作为脚本立即进行评估，并替换为其结果

您可以在运行脚本时传递参数。
参数将自动进行类型转换，并可以从数组[args]中引用。
\js[bbb,10,ccc] // 数组[10，'ccc']将存储在变量args中。

您也可以直接编写和嵌入脚本。
\js // 它将被脚本xxx的执行结果代替。
但是，如果在脚本中使用符号[<] [>]，则需要进行以下转义。
> : &gt;
< : &lt;

可以使用控制字符的情况如下。
・文字的显示
・备注栏（※）
・插件指令（※）
・插件参数（※）
・技能等的说明栏。
※仅限使用PluginCommonBase.js为基础导入的插件

通过使用此插件，您可以登录常用的脚本或文本并轻松调用它们。
即使数据库输入字段受到限制，也可以设置长字符串。

参数

文本基本列表 要登录的文本和脚本列表。

UniqueDataLoader

读取data文件夹下存在的任何json文件。
将json文件创建为可以Parse为JSON的文本文件。
游戏启动时将读取定义的文件。

数据将以指定的名称存储在全局对象中。
如果将「window」指定为全局变量名称，则每个对象都将被定义为全局变量，请注意名称冲突问题。

它可以用来读取数据库转换器MZ创建的数据或用其他单独插件添加的数据。

可以使用单独的插件或脚本如下所示读取加载的数据。

全局变量为[$dataUniques]属性名称为[property]时的参考示例
$dataUniques.property

成功读取所有固有数据后，将调用以下方法。
必要时请重新定义。
Scene_Boot.prototype.onUniqueDataLoad

参数

全局变量名 定义了固有数据的全局对象的名称。登录的变量将在该变量下生成。
数据列表 固有数据的列表。请将所有json文件配置在data文件夹下。
