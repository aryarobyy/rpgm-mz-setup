Map Event System

Map Event Processing Flow

  • Expressing Events

    The details of events expressed in the game using Map Events are defined using the following flow: where (location), what kind of situation (appearance conditions), what action is taken (trigger), and what happens (contents).

    Events such as characters and chests with items in them are expressed by assigning images to the Map Events which define the contents of what happens. Also, processes such as moving in and out of buildings, or cut-scenes that advance automatically, can be prepared using events that do not have images.

    When creating an event, you first need to decide "where (location) and what happens" in the map view of the Main Window. After that, you will be able to define the remaining contents such as the event's appearance conditions.

  • Event Pages with "Conditional" Contents

    It is possible to use Map Events to change the details of what happens according to the play conditions of the game.

    [Event Pages] are responsible for this. Event pages can "conditionally" determine the contents of events. One event may have up to 20 event pages, each one able to define different images, triggers, and what processes are to be run. In other words, you can "conditionally" choose from 20 different events in a single Map Event.

  • Restrict Event Pages Using Appearance Conditions

    The "conditions" parameter is used to define in "what situation" a "conditional" event page is to be used. Conditions are set based on criteria such as the values of "Switches" and "Variables" (see below), or "does the player have the specified item?"

    Map Events occur during gameplay based on the contents of the page which meet these conditions. When there are multiple pages which meet these [Conditions], the event page with the highest number will be used on the map. Conversely, nothing will happen on the map if there are no event pages which meet the [Conditions].

Evaluating Play Conditions

  • Evaluating Play Conditions Using Switches

    Under [Conditions] on the event page, [Switches] and [Variables] are primarily used for determining the play conditions of the game.

    Switches can have either an [ON] or [OFF] value. These are used to define conditions of the game and the state of progress such as "did the player accept the request from the king" or "did the player defeat the boss".

    The game will start with all switches set to [OFF]. When running an event that changes the position in the game, the specific switch's value in the "contents" will be switched to "ON". (Use the [Control Switches] event command.)

    By using that same switch in the [Conditions] of other event pages, it is possible to change the contents of multiple Map Events all at once when the switch's value changes to "ON". The game's story can be built through continuously changing the state of the overall game through the changing of these events.

  • Variables That Store Values

    "Variables" are what store arbitrary values (integers). The values at the start of the game are set to 0, and you can do calculations such as assigning values of your choice according to the state of the game or add to other values. (Use the [Control Variables] event command.)

    By using this variable, you can do things such as use the HP of the player or the amount of gold they possess to change the details of an event. This allows you to finely control a Map Event's conditions and results.

  • Management Switch Variables

    You may use up to 5,000 switches and 5,000 variables in one game. Which switch number or variable is used to evaluate each game state is up to the creator. Use names that help you understand their purposes so that you do not confuse the relationship of the two. You can set their names in the [Name] field in their respective selection windows.

  • Using the "Self Switch" for Autonomous Events

    You can also use "Self Switches" in Map Events. These switches are only used in their individual Map Event. You can store values in the 4 self switches, named "A" through "D", included in an event.

    This self switch is used for evaluating the state of events that are completed using self processing. For example, for treasure chests which contain items, by managing whether the chest was opened or not using self switches, and setting the contents for "before opening" and "after opening" in two event pages, you can make it so that an item can be obtained only the first time the chest was checked.
